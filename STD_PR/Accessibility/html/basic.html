Accessibility Basic

목록으로 이동

텍스트의 종류

기본텍스트 = 숨김텍스트 = 대체텍스트(alt) = 라벨텍스트(aria-label)

설명텍스트 = 용도, 목적, 경고에 대한 설명텍스트

숨김텍스트(blind)

  • 배경이미지 정보
  • 제목 정보 : 문서구조화를 위한 목차제공
    • 일반적인 논리적인 구조 : PC는 숨김텍스트, Mobile은 display:none (초점이슈가 발생)
    • 탭내용의 논리적인 구조 : PC는 숨김텍스트, Mobile은 display:none (초점이슈가 발생)

      - (권고) 목차로 제공할것은 권장하나, 탭 제목이 제공된 것으로 준수함.
      - 공식적인 WAI-ARIA 방법 : role="tabpanel" aria-labelledby="IDHREF"

      - Device Display 처리 예
      .dv-pc-only, .dv-mo-only, .dv-ios-only, .dv-and-only {display:none;}
      .dv-pc .dv-pc-only, .dv-ios .dv-ios-only {display:block;}

  • 부가 정보 : ("구매단가 : ") 구매단가 : 300,000
  • 상태 정보 : 선택됨, 현재항목 (예 : 모바일은 aria-selected, aria-current로 처리가능, PC는 숨김텍스트 처리)
    1. 처음부터 제공하고 필요한것 활성화
    2. 필요할때 append로 제공해서 활성화

대체텍스트(alt)

  • 의미없는 이미지 : alt="[빈값]"
  • 텍스트 이미지 : alt="[텍스트]"
  • 정보성 이미지 : alt="[텍스트] + [구분]"
    [텍스트]
    이름 또는 설명
    [구분]
    로고, 앱 아이콘, 썸네일, 대표사진 (한글로 표현할 것을 권장)

라벨텍스트(aria-label)

  • 기본텍스트 무시하고 aria-label로 읽어준다. (단, 기본텍스트가 생략될 수 없다.)
  • 특수문자의 대체텍스트 : * (aria-label="필수입력")
  • 원문에대한 부가정보 : 300,000 (aria-label="구매단가 : 300,000")

설명텍스트(title)

  • 새창열림 : title="새창열림"
  • 더보기 : title="공지사항 더보기"
  • 확인 : title="가입완료 확인", title="메인페이지로 이동" aria-label="가입완료 확인"
  • 닫기 : title="만족도팝업 닫기"
  • 제목링크 : title="공지사항 상세페이지로 이동"
  • 1단계 : title="가입 1단계 동의하기"

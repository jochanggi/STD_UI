[코드규칙]
- 주석
	<!-- s: 시작하는주석 -->
	<!-- e: 종료하는주석 -->
	<!-- Dev 개발팀 참고사항 -->
	<!-- Pub 퍼블팀 참고사항 -->
	<!-- All 모든팀 참고사항 -->
	<!-- 수정 날짜 내용 -->
	<!-- 추가 날짜 내용 -->
	<!-- 삭제 날짜 내용 
		(삭제될 코드)
	-->
	<!-- 고도화1 날짜 내용 -->
	<!-- 고도화2 날짜 내용 -->
	<!-- 고도화3 날짜 내용 -->

- 수정, 추가, 삭제는 단건 조치에 대한 기록을 남기는 용도
- 고도화123은 다건 조치에 대한 단위별로 기록을 남기는 용도
- 조치범위가 긴경우 s: e: (시작,종료) 주석이 같이 사용됨)


[마스터 레이아웃]
- 템플릿은 .content 부터 시작한다.
- 팝업은 .content 내에 포함한다. 단, 어느위치에서도 문제되지 않아야 한다. (앱개발 구조를 고려)


[구조화 레이아웃]
- 레이아웃 요소와 시맨틱요소를 분리한다.
- 형제관계의 간격을 자식구조가 관여하지 않는다.
- 레이아웃에 꾸미기의 스타일을 적용하지 않는다.


[컨텐츠 레이아웃]
- 컨텐츠 섹션간 레이아웃은 페이지로 모듈화 하고 제어한다.
- UI 형제간 레이아웃은 컨텐츠로 모듈화 하고 제어한다.


[UI]
- 버튼
	텍스트 있는 버튼 : <button type="button" class="btn btn-타입">버튼</button>
	아이콘 전용 버튼 : <button type="button" class="btn-ico btn-이름"><i class="ico ico-이름"></i></button>


- 아이콘
	블릿 아이콘 : <i class="bul bul-이름"></i>
	방향 아이콘 : <i class="arw arw-이름"></i>
	일반 아이콘 : <i class="ico ico-이름"></i>
